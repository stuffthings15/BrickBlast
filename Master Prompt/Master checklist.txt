GAMEPLAY REQUIREMENTS:
Implement a full playable loop:

1. Player opens the game.
2. Main menu appears.
3. Player taps Play.
4. Current level loads.
5. Ball is ready to launch.
6. Player controls paddle with touch / mouse.
7. Player launches ball.
8. Ball destroys bricks.
9. Player earns score and currency.
10. Level completes when all required bricks are destroyed.
11. Game over occurs when all lives / balls are lost.
12. Results screen displays performance.
13. Player can continue, replay, visit marketplace, or return to menu.
14. Progress is saved.
15. Marketplace purchases persist.
16. Network sync can upload or retrieve profile / leaderboard / save metadata.

BRICK TYPES:
Implement at least five brick types:

1. Standard Brick
- Breaks in one hit.
- Grants base score.

2. Durable Brick
- Requires multiple hits.
- Changes visual state as health decreases.

3. Reward Brick
- Grants extra currency or bonus score.
- May spawn a reward effect.

4. Hazard Brick
- Adds risk.
- May speed up the ball, reduce paddle size temporarily, or trigger a negative effect.

5. Moving Brick or Shielded Brick
- Moving Brick moves within a lane.
- Shielded Brick requires condition before full damage.
- Pick one and implement it fully.
